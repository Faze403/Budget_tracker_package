# 자기 평가서

## 1. 가장 잘했다고 생각하는 부분

이번 프로젝트에서 가장 잘했다고 생각하는 부분은 테스트를 비교적 일찍부터 작성하기 시작한 점이다.
정상 케이스뿐 아니라 날짜 형식 오류, 잘못된 금액 타입, 빈 거래 목록, 잘못된 연월 문자열 같은 경우도 함께 다뤄서, 기능을 추가할 때 기존 동작이 깨지지 않도록 확인할 수 있었다.

## 2. 아쉬운 부분

아쉬운 점은 현재 구현이 메모리 기반이라 프로그램을 종료하면 데이터가 유지되지 않는다는 것이다.
과제 범위 안에서 수입/지출 기록, 달별 조회, 리포트 생성까지는 구현했지만, JSON 파일 저장과 불러오기 기능까지 포함했더라면 실제 사용성이 더 높아졌을 것이다. 또한 사용자가 더 편하게 실행할 수 있는 CLI 인터페이스를 구현하지 못한 점도 아쉽다.

## 3. 다음에 개선하고 싶은 점

다음에 개선한다면 우선 거래 데이터를 파일로 저장하고 다시 불러오는 기능을 추가하고 싶다.
JSON 저장/불러오기를 구현하면 지금 만든 구조를 실제 가계부처럼 계속 사용할 수 있고, 테스트도 더 현실적인 방향으로 확장할 수 있을 것이다.
그리고 월간 리포트를 문자열뿐 아니라 구조화된 데이터로도 반환하도록 바꿔서, 다른 출력 형식으로 확장하기 쉬운 구조를 만들면 좋을 것 같다.

## 4. AI 도구 사용 내역

이번 프로젝트에서는 Codex/ChatGPT와 Claude를 보조적으로 활용했다.
프로젝트 구조를 단계별로 나누는 계획 수립, 클래스와 테스트 초안 작성, 영문 예외/커밋 메시지, docstring 정리 등의 과정에서 도움을 받았다.
다만 생성된 결과를 그대로 제출하지 않고, 수업 범위에 맞는지와 동작 방식을 직접 확인했다. 또한 표현과 테스트 데이터를 직접 수정하고 테스트를 다시 실행하며 필요한 부분을 재검토했다.
